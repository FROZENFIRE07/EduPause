### Core Design Decisions We Finalized

1. **Hybrid JS + Python architecture**  
   - 80–90% MERN (familiar)  
   - Intelligent core (agents, loops, stateful orchestration, break recovery) → Python LangGraph (smoother cycles, checkpointing to MongoDB Atlas, production-like patterns)

2. **Ingestion pipeline order (safer & higher quality)**  
   Raw transcript → **Chunk-level summarization** → **Chunk summaries used for**:
   - Concept / prerequisite extraction → Neo4j KG
   - Embedding & vector storage → Qdrant/Pinecone
   - Agent context (tutor, planner, evaluator)

   → **Optional** playlist-level (reduce) summary for high-level overview & break recovery priming

3. **Summarizer prompts style**  
   - Rich, detailed, educational tone  
   - Preserve technical precision (formulas, steps, definitions)  
   - Note prerequisites explicitly  
   - Include approximate timestamps for anchors  
   - Target 20–35% length of chunk

4. **Agent roles & flow** (LangGraph nodes)  
   - **Observer** → analyzes clickstream → computes probabilistic confusion score  
   - **Planner** → queries Neo4j KG + mastery state → suggests next chunk/video  
   - **Socratic Tutor** → Groq prompt → question / hint / analogy (based on current summary + timestamp)  
   - **Evaluator** → scores user answer → updates mastery in MongoDB  
   - **Orchestrator / graph** → conditional routing + cycles (tutor → evaluator → observer until mastery)  
   - **Break Recovery entry point** → pre-exposure priming + recap summary + low-stakes warm-up questions

5. **Clickstream heuristics**  
   - Weighted signals: rewind density, short pauses, speed reduction, click frequency  
   - Probabilistic score (not rigid thresholds)  
   - Distinguish cognitive impasse vs external distraction

6. **Persistence & resumability**  
   - LangGraph checkpointer → MongoDB Atlas (shared with Node.js)  
   - Thread_id = user session ID → resume exactly where left off (even after weeks)
